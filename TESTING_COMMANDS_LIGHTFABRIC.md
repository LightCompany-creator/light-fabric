## 🛡 ПОДГОТОВКА (сделай ОБЯЗАТЕЛЬНО перед тестированием)

В терминале в папке LightFabric:

```bash
# 1. Сохрани текущее состояние — это страховка
git add .
git commit -m "checkpoint: stable state before full testing"
git log --oneline -5  # увидишь последние 5 коммитов для отката

# 2. Создай отдельную ветку для тестирования
git checkout -b testing/full-audit

# 3. Убедись что dev-сервер запущен
npm run dev
# Должен открыться http://localhost:3000
```

Если что-то пойдёт не так — вернёшься на main одной командой:
```bash
git checkout main
```

## 💡 ВАЖНЫЕ ПРИНЦИПЫ

**1. Один аудит — один отчёт — починка по очереди.**
Не давай Claude Code «найди и почини всё» — будет хаос. Сначала отчёт, потом починка по одной.

**2. Git коммитов много, не жалей.**
После каждой починки — коммит. Если что-то сломается — откатишься без боли.

**3. Доверяй коду, не плану.**
Проект уже разошёлся с WORKPLAN.md. Это нормально. Команды выше работают от того, что реально есть в коде.

**4. UX проверяет только человек.**
AI отлично проверит технику. Но удобно ли бригадиру нажимать кнопки грязными руками в цеху — проверит только сам бригадир. После технического тестирования — обязательно поход в цех.

**5. Не торопись с production.**
Если хоть один пункт безопасности или надёжности красный — запуск переносится. Лучше неделя задержки чем потеря данных в первый день.
