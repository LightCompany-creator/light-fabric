# SETUP — развёртывание LightFabric (для разработчика)

> Для Арсена (1С-программист Light Company) и любого разработчика,
> кому нужно поднять систему у себя и посмотреть функционал.
> Это актуальный источник истины по запуску (README устарел частично).

LightFabric — MES-система цехового учёта: смены, выработка, сдельная ЗП,
расход материалов, перемещения между цехами.

- **Стек:** Next.js 14 (App Router) + TypeScript + Tailwind + shadcn/ui
- **БД/Auth:** Supabase (Postgres + Auth + RLS)
- **Node.js:** 18+ (рекомендуется 20 LTS)

---

## Вариант А — посмотреть функционал быстро (рекомендуется)

Подключиться к **уже настроенной облачной базе** с демо-данными и тестовыми
аккаунтами. Ничего в БД настраивать не нужно.

1. Клонировать репозиторий и поставить зависимости:
   ```bash
   git clone https://github.com/LightCompany-creator/light-fabric.git
   cd light-fabric
   npm install
   ```
2. Создать файл `.env.local` (Наира пришлёт значения отдельно — в репозиторий
   они не коммитятся):
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://<ref>.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=<anon-public-key>
   ```
   > `anon`-ключ публичный (он и так уходит в браузер, доступ ограничен RLS).
   > `SERVICE_ROLE_KEY` для просмотра функционала НЕ нужен.
3. Запустить:
   ```bash
   npm run dev
   ```
4. Открыть `http://localhost:3000/demo` — карточки ролей с входом в один клик.
   Или вручную: `admin@lightflow.test` / `Test123!` (все роли — пароль `Test123!`).

Роли для обзора: `foreman` (начальник цеха), `technologist` (справочники),
`director` (дашборд/отчёты), `accountant` (ЗП, остатки, обмен с 1С), `admin` (всё).

## Вариант Б — поднять свою базу с нуля

Если нужна полностью своя инсталляция (например, на сервере Light Company
212.113.123.188).

1. Клонировать + `npm install` (как выше).
2. Создать проект на supabase.com (или self-hosted Supabase).
3. В **SQL Editor** выполнить целиком `schema.sql` (базовые таблицы + артикулы).
4. Заполнить `.env.local` (см. `.env.example`) — все 4 переменных, включая
   `SUPABASE_SERVICE_ROLE_KEY` и `DATABASE_URL` (нужны для скриптов/миграций).
5. Накатить миграции (поверх schema.sql):
   ```bash
   node --env-file=.env.local scripts/run-migrations.mjs
   ```
   Это применит `supabase/migrations/0002…0012` по порядку
   (RLS, переименования цехов, удаление партий, склады/перемещения, демо-RLS).
6. Создать тестовые аккаунты и наполнить демо-данными:
   ```bash
   node --env-file=.env.local scripts/seed-users.mjs            # 5 ролей @lightflow.test
   node --env-file=.env.local scripts/seed-workshop-foremen.mjs # начальники цехов
   node --env-file=.env.local scripts/seed-demo-shifts.mjs      # 11 демо-смен за 2 недели
   ```
7. `npm run dev` → `http://localhost:3000/demo`.

> Импорт реальных справочников (артикулы, расценки) — скрипты
> `import-data.mjs`, `import-rates.mjs`/`apply-rates.mjs`. Для демо не обязательны.

## Что смотреть (основной функционал)

- **Смены** — открытие/закрытие (день/ночь), выработка по работникам и операциям,
  авто-расчёт сдельной ЗП на каждое изменение.
- **Справочники** — артикулы (388), расценки (610), сотрудники, цеха (11).
- **Перемещения между цехами** — документ передачи: отправитель создаёт →
  получатель принимает → остаток списывается/приходуется → документ закрывается.
- **Остатки** (`/stocks`) — склад у каждого цеха.
- **Дашборд директора** — ФОТ, производство, материалы; period picker; XLSX-экспорт.
- **PWA** — установка на iPad/Android, офлайн-очередь мутаций.

Интеграция с 1С (для Арсена): спецификация обмена — в папке `1С-Арсену/`.

## Полезные команды

| Команда | Что делает |
|---|---|
| `npm run dev` | Dev-сервер на :3000 (HMR) |
| `npm run build` | Прод-билд (проверка TypeScript + ESLint) |
| `npm run start` | Запуск прод-билда |
| `npm run lint` | Только ESLint |
| `node --env-file=.env.local scripts/e2e-shift-flow.mjs` | E2E: открыть смену → выработка → закрыть → проверить ЗП |

> Не запускать `npm run build` пока работает `npm run dev` — может сломать кэш `.next`.

## Известные нюансы

- **RLS расслаблены для демо** (миграция 0011): любой работник пишет в любой цех.
  Перед боевым запуском вернуть строгие правила.
- **Партионного учёта нет** (удалён миграцией 0009) — учёт идёт через смены.
- **Next.js 14.2.35** — перед продом обновить до 14.2.40+ (есть CVE).
