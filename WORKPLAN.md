## 🎯 НЕДЕЛЯ 3 — Главная фича: смены литейки (Claude Code доминирует)

**Это сердце системы. Здесь точная логика критична.**

### Сервис смен 🧠
- [ ] **Промпт Claude Code:** «Создай сервис `lib/services/shifts.ts` с функциями:
  - `openShift(workshopId, foremanId)` — открывает смену, проверяет нет ли уже открытой
  - `addOutput(shiftId, articleId, qty, weight, defect, machine, downtime)` — добавляет выработку
  - `addWorker(shiftId, employeeId, operations)` — добавляет работника в смену
  - `closeShift(shiftId)` — закрывает смену:
    1. Проверяет валидность данных
    2. Создаёт партии из каждой записи shift_outputs
    3. Считает зарплату каждого работника через сервис payroll
    4. Списывает сырьё по нормам через сервис materials
    5. Обновляет status='closed'
  Все операции в транзакции. Покрой Zod-схемами.»

### Сервис расчёта зарплаты 🧠
- [ ] **Промпт Claude Code:** «Создай сервис `lib/services/payroll.ts` с функцией `calculateShiftPay(shiftId)`:
  1. Получает все shift_outputs и shift_workers смены
  2. Для каждого работника проходит по его операциям
  3. Находит актуальную rate из таблицы rates (с учётом article_id или общую цеховую)
  4. Считает: qty × rate
  5. Сохраняет в shift_workers.calculated_pay
  6. Возвращает summary.
  Учитывай valid_from/valid_to расценок. Если расценка не найдена — лог ошибки в shift.notes.»

### Сервис формирования партий 🧠
- [ ] **Промпт Claude Code:** «Создай сервис `lib/services/batches.ts` с функциями:
  - `createBatchFromOutput(outputId)` — создаёт партию из записи выработки:
    1. Генерирует batch_number через функцию БД generate_batch_number
    2. Создаёт запись в batches
    3. Создаёт первую запись batch_movements (from=null, to=цех создателя)
    4. Связывает с shift_output (batch_id)
  - `receiveBatch(batchId, workshopId, employeeId, qtyIn, defect)` — приёмка партии
  - `transferBatch(batchId, toWorkshopId, employeeId)` — передача в следующий цех

## 🛠 Шпаргалка по работе с AI

### Когда зовём 🧠 Claude Code
- Архитектурные решения, новые модули
- Сложная бизнес-логика (зарплата, маршруты, выгрузки)
- Многошаговые задачи с несколькими файлами
- Когда нужно понять контекст всего проекта
- Рефакторинг и оптимизация

### Когда зовём ⚡ Codex
- Один компонент по образцу
- CRUD-страницы по шаблону
- Простые формы и таблицы
- Стилизация, респонсив
- Юнит-тесты

### Стандартный промпт для любой задачи
```
Прочитай PROJECT_CONTEXT.md и WORKPLAN.md (раздел про эту задачу).
Задача: [описание]
Ограничения:
- TypeScript строгий
- Только shadcn/ui для UI
- БД через Supabase client
- Цвета из brand palette
- Бизнес-логика в lib/services/
Перед кодом — план действий, жду подтверждения.
```

### Что сделать ПОСЛЕ каждой задачи
1. Запустить `npm run typecheck` — нет ошибок?
2. Запустить `npm run lint` — нет warnings?
3. Открыть страницу в браузере — работает?
4. `git add . && git commit -m "..."`
5. Отметить задачу `[x]` в этом файле
