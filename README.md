## ⚡ Быстрый старт (за вечер)

### Шаг 1: Заведи аккаунты (15 минут)
- **GitHub** — github.com (для хранения кода)
- **Supabase** — supabase.com (база данных, бесплатный тариф)
- **Vercel** — vercel.com (деплой, бесплатный тариф)
- **Claude Code** — claude.ai/code (или установи через `npm install -g @anthropic-ai/claude-code`)
- **Codex** — через ChatGPT Plus или OpenAI API

### Шаг 2: Создай проект в Supabase (10 минут)
1. На supabase.com нажми **New Project**
2. Имя: `lightflow`, регион: ближайший
3. Сохрани пароль БД — он понадобится
4. Дождись готовности (~2 минуты)
5. Зайди в **SQL Editor** → **New query**
6. Скопируй содержимое `schema.sql` целиком → вставь → **Run**
7. Проверь в **Table Editor**: должно быть 15 таблиц, в `articles` — 46 записей

### Шаг 3: Создай папку проекта (5 минут)
```bash
# В удобной папке
mkdir lightflow
cd lightflow

# Положи сюда все 4 файла из этого пакета
# (PROJECT_CONTEXT.md, schema.sql, WORKPLAN.md, README.md)

# Инициализируй git
git init
git add .
git commit -m "init: project specification"

# Создай репозиторий на GitHub и связкой:
git remote add origin git@github.com:USERNAME/lightflow.git
git push -u origin main
```

### Шаг 4: Запусти Claude Code (5 минут)
В терминале в папке проекта:
```bash
claude-code
```

Первый промпт:
```
Прочитай PROJECT_CONTEXT.md и WORKPLAN.md (раздел НЕДЕЛЯ 1).
Начни с задачи "Инициализация проекта" — создай Next.js 14 с TypeScript,
Tailwind, Supabase, shadcn/ui. Структура папок по разделу 12 контекста.

Перед кодом — план действий, жду подтверждения.
```

### Шаг 5: Дальше по WORKPLAN.md
Открой `WORKPLAN.md`, иди по задачам сверху вниз. Каждая задача отмечена:
- 🧠 **Claude Code** — сложные задачи, архитектура
- ⚡ **Codex** — простые компоненты, формы
- 👥 **Оба** — совместная работа

После каждой задачи: коммит в git, отметка `[x]` в WORKPLAN.

## 🖥 Локальный запуск (после Шагов 1–5)

```powershell
# Установка зависимостей (один раз)
npm install

# 1) Положи в .env.local URL и ключи Supabase (см. .env.example)

# 2) Запусти dev-сервер
npm run dev
```

Открой `http://localhost:3000`. Логин одним из тестовых пользователей —
`foreman@lightflow.test` / `Test123!` (см. скрипт `scripts/seed-users.mjs`).

### Создать тестовых пользователей

```powershell
node --env-file=.env.local scripts/seed-users.mjs
```

Скрипт создаёт 4 аккаунта (foreman, technologist, director, accountant)
и связывает их с записями в `employees`.

### Применить RLS-миграцию

В Supabase Dashboard → **SQL Editor** → **New query** → вставить целиком
`supabase/migrations/0002_rls_minimal.sql` → **Run**. Без этого технолог
не сможет редактировать справочники, а бригадир — открывать смены.

## 🚀 Деплой на Vercel

### Через интерфейс

1. Запушь репозиторий на GitHub.
2. На `vercel.com` нажми **Add New… → Project**, выбери репо.
3. Framework — **Next.js** (определится автоматически).
4. **Environment Variables** — добавь те же, что в `.env.local`:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY` (нужен только для админ-скриптов в проде)
5. **Deploy** — через ~1 минуту получишь preview-домен `*.vercel.app`.
6. **Production** деплой запускается автоматически на push в `main`.

### Свой домен (например `mes.light-c.shop`)

1. В Vercel → проект → **Settings → Domains** → добавь домен.
2. У регистратора домена пропиши CNAME `mes.light-c.shop → cname.vercel-dns.com`.
3. Через 5–30 минут домен подключится с HTTPS-сертификатом.

### Обязательная проверка прода

- Залогиниться тестовым пользователем
- Открыть смену, добавить выработку, закрыть
- Проверить что партия создалась
- Скачать XLSX-ведомость из `/reports/payroll`
