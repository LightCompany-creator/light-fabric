# Сценарий демонстрации LightFabric

**URL:** https://light-fabric.vercel.app
**Длительность:** 12-15 минут
**Аудитория:** директор, начальник производства, бригадиры, технолог, бухгалтер

---

## 🎬 Перед началом

### Подготовка устройства
- [ ] Планшет (iPad или Android, ширина не меньше 768px) или ноутбук
- [ ] Wi-Fi проверен — `https://light-fabric.vercel.app` открывается
- [ ] При желании установлено как PWA (Safari → Поделиться → На экран «Домой», Chrome → меню → Установить)
- [ ] Зарядка ≥50%

### Что в системе уже есть
- 11 цехов из реальной структуры Light Company
- 388 артикулов с light-c.shop (коды, названия, цены, материалы)
- 102 работника из вашего списка
- 610 расценок по фото от технолога
- **11 закрытых смен за последние 2 недели** — для показа дашборда и отчётов
- Средние веса пар для 10 ходовых артикулов (автоподстановка в форме)

### Что пока «по умолчанию» (что хотим докрутить с производством)
- Должности у 74 сотрудников из 102
- Нормы расхода материалов у 342 артикулов из 388
- Веса пар у 378 артикулов из 388
- Реальные остатки сырья на складе

> Эти пробелы — ваша демонстрация **«вот что система знает, вот что мы у вас попросим»**.

## ⏱ Скрипт по минутам

### [0:00–1:00] Вступление и установка ожиданий

> «Это LightFabric — система оперативного учёта производства Light Company.
> Заменяет тетради и Excel-таблицы бригадиров: смены, выработка, сдельная ЗП,
> расход сырья. Сейчас покажу за 12 минут от лица 4 ролей: бригадира,
> технолога, директора и бухгалтера.»

Открыть **https://light-fabric.vercel.app**

---

### [1:00–4:00] 🟢 Роль 1: Бригадир Литейки (ключевая роль)

Зайти как `foreman@lightflow.test` / `Test123!` (или клик на «Литейку» в /demo).

**Покажите:**

1. **Левое меню** — для бригадира только то, что ему нужно: Главная, Смены, Инструкция на стену
2. **Главная** — карточка «Смена не открыта» + кнопка «+ Открыть смену»
3. **Откройте смену** на сегодня → тип «дневная» → автоматически выберется Литейка → нажмите «Открыть смену»
4. **Добавьте выработку** (1-2 записи):
   - Артикул: `007` (Обувь мужская, Туфли купальные)
   - **Вид:** 6-парка
   - **Пар:** 50
   - **Кг:** заполнилось автоматически 12.00 ← ⭐ показать это («вес считается сам, бригадиру не надо умножать в голове или искать калькулятор»)
   - Брак: 1
   - Машина: KCLKA 1
   - Простой: 0
   - «Добавить» → запись появилась в таблице снизу
5. **Добавьте работника** (блок «Работники в смене» → «+ операция»):
   - Работник: любой из списка (например, «Аминов Атохон Баяндикович, 0016»)
   - Артикул: `007`
   - Операция: `литьё`
   - Кол-во: 50
   - «+» → работник появится в таблице с показателем выработки
6. **Покажите оффлайн** (опционально, если есть тестовый режим):
   - Включите авиарежим на планшете → добавьте ещё запись → появится баннер «Сети нет — запись отложена»
   - Сверху индикатор «Нет связи»
   - Выключите авиарежим → через несколько секунд индикатор станет «Синхронизация», потом исчезнет
   - «Бригадир может работать даже когда Wi-Fi пропал — данные не теряются»
7. **Закрыть смену** (кнопка справа сверху) → подтвердить
   - «Система автоматически: посчитала ЗП работнику, списала сырьё по нормам, перевела смену в архив»

**Главная мысль:** «Бригадиру не нужно ничего считать. Он только вводит «что» и «сколько». Всё остальное система делает сама.»

---

### [4:00–6:00] 🔵 Роль 2: Технолог (как ведутся справочники)

Выйти → войти как `tech@lightflow.test` / `Test123!`.

**Покажите:**

1. **Меню технолога** — Артикулы, Расценки, Нормы, Работники, Материалы, Цеха
2. **Артикулы** → 388 позиций с light-c.shop → откройте любой (например, `007`)
   - Поля: код, название, материал, размеры, в коробке, вес пары
   - «Это уже залито из вашего сайта-каталога — обновляется через XLSX-импорт из 1С»
3. **Расценки** → 600+ строк по цехам и артикулам
   - «Сюда технолог заносит ставки сдельной оплаты — по фото от вас уже 600 расценок»
4. **Нормы** → 47 записей
   - **Здесь скажите:** «У нас сейчас нормы только на часть артикулов. Чтобы система точно списывала ЭВА со склада, нужно от технолога: на каждый артикул — сколько грамм/кг какого материала уходит на пару. Шаблон я подготовила — заполняется в Excel.»
5. **Работники** → 102 человека → откройте любого → можно сменить цех, дать роль, поставить должность

**Главная мысль:** «Технолог — единая точка правды для всего, что про "как делается". Расценки, нормы, маршруты, артикулы — всё здесь.»

---

### [6:00–9:00] 🟠 Роль 3: Директор (зачем всё это)

Выйти → войти как `director@lightflow.test` / `Test123!`.

**Покажите:**

1. **Дашборд** — главное:
   - KPI сверху: пар сегодня, выпуск в рублях, брак %, ФОТ за день, расход ЭВА в кг
   - **Топ артикулов за неделю** — реальные позиции (007, 022/1, 112/н...)
   - **Выпуск по дням за 2 недели** — линейный график (~3000 пар за период)
   - **Загрузка цехов сегодня** — столбики по 11 цехам
2. **Все смены** (меню → Все смены) → 11 строк за май
   - Откройте любую — увидите выработку, работников, посчитанную ЗП
3. **Производство** (меню → Производство) → отчёт за период
4. **Ведомость ЗП** → за май → ~14 000 ₽ начислено, можно скачать XLSX

**Главная мысль:** «Раньше эти цифры собирались вручную в конце месяца. Сейчас директор видит "сегодня" в реальном времени, и любую глубину детализации — одна-две клика.»

---

### [9:00–11:00] 🟡 Роль 4: Бухгалтер (обмен с 1С)

Выйти → войти как `accountant@lightflow.test` / `Test123!`.

**Покажите:**

1. **Меню бухгалтера** — только то, что ему нужно: ЗП, выпуск, расход, Обмен с 1С
2. **Ведомость ЗП** → выберите май → «Скачать XLSX» → откройте файл
   - Внутри: табельный, ФИО, цех, сумма, период — готовый формат для 1С
3. **Обмен с 1С → Импорт** → загрузить XLSX с новыми артикулами или сотрудниками
4. **Обмен с 1С → Журнал** → видны все импорты/экспорты с количеством и статусом

**Главная мысль:** «Связка с вашей 1С Бухгалтерией — через файлы. Никаких интеграций по API не нужно настраивать. Удобно для аудита.»

### [11:00–12:30] 📱 Бонус: «А это работает на планшете в цеху?»

**Главная мысль:** «Это полноценное приложение для планшета или телефона. Бригадир в цеху работает на iPad как с обычной программой.»

### [12:30–15:00] 💬 Q&A и сбор «хотелок»

Заведите блокнот / документ — записывайте всё, что услышите:
- «А можно добавить колонку X?»

- «А мы хотим, чтобы бригадир видел/не видел W»

Большинство правок занимают 5-30 минут. Архитектура поддерживает гибкость.

## 🆘 На случай если что-то пойдёт не так

### Если приложение не открывается на планшете
- Проверь Wi-Fi
- Открой `https://light-fabric.vercel.app` в браузере телефона как фолбэк
- Скриншот ошибки сохрани — починим после демо

### Если кнопка не нажимается / форма не сохраняет
- Перезагрузи страницу (pull-to-refresh или Ctrl+R)
- Если упорно — открой в режиме инкогнито (там нет кэша)

### Если случайно «закрыла смену» не ту
- Не критично — данные сохранились
- В прод-режиме откатим, в демо — оставим, ничего не сломалось

### Если в списке нет нужного артикула или работника
- Это значит данных в БД нет — пометь себе, добавим после демо
