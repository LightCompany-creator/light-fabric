## Архитектурные решения, которые приняли

1. **Партионный учёт удалён** (миграция 0009). Тогда был запрос «мы не ведём учёт по партиям». Удалены таблицы `batches`, `batch_movements`, `routes`. Колонки `articles.route_type`, `shift_outputs.batch_id`.
2. **Поле «Вид»** (4-парка/6-парка) на `shift_outputs` — миграция 0010, `cast_forms int`.
3. **RLS расслаблены для демо** (миграция 0011) — любой работник с ролью открывает смену в любом цехе. Перед боевым запуском — вернуть строгие правила.
4. **PWA-приложение** (фазы 4-5): иконки, service worker, install prompt, splash screens iOS, оффлайн-очередь для addOutput и addWorker через localStorage с auto-replay по событию 'online'.
5. **Демо-страница `/demo`** — one-click вход в любую из 5 ролей.
6. **Date range picker** на дашборде директора — кнопки Сегодня/Неделя/Месяц/Произвольно + URL searchParams.
7. **Автоподстановка веса** в форме выработки: `quantity × weight_per_pair`.
8. **Автоматический пересчёт ЗП** на каждый `addOutput`/`addWorker`/`closeShift` (commit 102ff70 — last fix).
9. **Операции — выпадающий список** в форме «Работники в смене» (взято из rates цеха).
10. **Цеха на дашборде** показываются по-русски (`w.name`) с углом -35° (bar-chart.tsx).

## Важные скрипты (в `scripts/`)

| Скрипт | Назначение |
|---|---|
| `run-migrations.mjs` | Прогон SQL-миграций через DATABASE_URL (нужен в .env.local) |
| `seed-users.mjs` | Создаёт 5 тестовых auth-аккаунтов |
| `import-data.mjs` | Импорт артикулов + сотрудников из XLSX через service_role (минует RLS) |
| `apply-rates.mjs` | Расценки из фото (литейка/обрезка/упаковка для ЭВА и ПВХ) |
| `seed-demo-shifts.mjs` | 11 тестовых смен за 2 недели для оживления дашборда |
| `make-data-templates.mjs` | Генерит 6 XLSX-шаблонов в `data-templates/` |
| `generate-splash.mjs` | iOS splash screens (требует `sharp`) |
| `e2e-shift-flow.mjs` | E2E-тест: открыть смену → выработка → работник → закрыть → проверить ЗП |

## Поведенческие предпочтения Наиры

- **Не задавай много уточняющих вопросов** — действуй с разумными дефолтами.
- **«Сделай как считаешь правильным»** = дай рекомендацию и выполни, не откатывайся к обсуждению.

- **Делать → коммитить → деплоить** в один заход когда задача завершена.
- **Не врать про "ультра-надёжность"** — честно говорить что готово, что нет.
