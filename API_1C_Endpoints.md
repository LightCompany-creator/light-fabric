# Список endpoint'ов HTTP-сервиса 1С для LightFabric

> Для Арсена (1С-программист Light Company).
> LightFabric — программа цехового учёта (смены, выработка, сдельная ЗП).
> 1С (БП 3.0, 8.3.27) — мастер справочников и получатель итоговых документов.
> Составлено: 11 июня 2026.

---

## Общие договорённости

- **Транспорт:** HTTP-сервис в расширении 1С, JSON, UTF-8, HTTPS.
- **Авторизация:** технический пользователь 1С, Basic Auth (или токен в заголовке — как тебе удобнее).
- **Даты:** ISO 8601 (`2026-06-11` / `2026-06-11T14:30:00`).
- **Ключ состыковки:** внутренний **код 1С** у каждого элемента справочника. Мы храним его у себя и всегда передаём в документах.
- **Идемпотентность:** в каждом POST мы передаём `external_id` (UUID документа в LightFabric). Повторный POST с тем же `external_id` не должен создавать дубль — вернуть номер уже созданного документа.
- **Ответ на POST:** `{ "ok": true, "doc_number": "ЛФ-00000123", "doc_date": "..." }` либо `{ "ok": false, "error": "текст" }`.
- **Инкрементальная выгрузка:** во всех GET-справочниках параметр `?changed_since=<datetime>` — отдавать только изменённые с этого момента. Без параметра — полная выгрузка.
- **Партионного учёта нет** — никакие партии/серии в обмене не участвуют.

Префикс ниже условный: `/lightfabric/hs/api/v1/…` — назови как принято у вас.

## Этап 1. Справочники: 1С → LightFabric (GET)

### 1. `GET /nomenclature?changed_since=`
Номенклатура (товары производства + материалы/сырьё).
```json
[{
  "code": "00-00001234",
  "article": "112/н",
  "name": "Сапоги женские ЭВА с манжетой",
  "type": "Товар | Материал",
  "unit": "пар | кг | м | шт",
  "group": "Готовая продукция/Обувь",
  "is_archived": false
}]
```

### 2. `GET /employees?changed_since=`
Сотрудники (физлица, работающие в производстве).
```json
[{
  "code": "00-0000056",
  "tab_number": "0056",
  "full_name": "Иванова Анна Петровна",
  "department_code": "00-000003",
  "position": "Литейщик",
  "is_active": true
}]
```

### 3. `GET /departments`
Подразделения — сопоставим с нашими цехами вручную один раз.
```json
[{ "code": "00-000003", "name": "Литейный цех", "parent_code": null }]
```

### 4. `GET /warehouses`
Склады.
```json
[{ "code": "00-000001", "name": "Склад сырья", "is_active": true }]
```

### 5. `GET /organizations`
Организации (для шапки документов).
```json
[{ "code": "00-000001", "name": "ИП Хачатуров А.В.", "inn": "262800000688" }]
```

### 6. `GET /stock?warehouse_code=` *(желательно, не блокер)*
Остатки номенклатуры на складе — для сверки наших остатков с 1С.
```json
[{ "nomenclature_code": "00-00001234", "qty": 1250.5, "unit": "кг" }]
```

## Этап 2. Документы: LightFabric → 1С (POST)

### 7. `POST /production-report`
Отчёт производства за смену (по закрытию смены у нас).
```json
{
  "external_id": "uuid",
  "date": "2026-06-11",
  "organization_code": "00-000001",
  "department_code": "00-000003",
  "warehouse_code": "00-000002",
  "lines": [
    { "nomenclature_code": "00-00001234", "qty": 640, "unit": "пар" }
  ],
  "comment": "Смена дневная, ЛФ №2026-06-11-LIT-Д"
}
```

### 8. `POST /material-consumption`
Требование-накладная — списание сырья/материалов за смену.
```json
{
  "external_id": "uuid",
  "date": "2026-06-11",
  "organization_code": "00-000001",
  "department_code": "00-000003",
  "warehouse_code": "00-000001",
  "lines": [
    { "nomenclature_code": "00-00000077", "qty": 410.0, "unit": "кг" }
  ]
}
```

### 9. `POST /payroll`
Начисление сдельной ЗП (по закрытию месяца, суммы уже посчитаны у нас).
```json
{
  "external_id": "uuid",
  "period": "2026-06",
  "organization_code": "00-000001",
  "lines": [
    { "employee_code": "00-0000056", "amount": 48250.00, "calc_type": "Сдельная оплата" }
  ]
}
```
❓ Вопрос: каким видом расчёта/документом вам удобнее принимать сдельную ЗП в БП 3.0 — подскажи, подстроимся.

### 10. `POST /goods-transfer` *(этап 2+, обсудим)*
Перемещение готовой продукции на склад ГП (если решим отражать в 1С).
```json
{
  "external_id": "uuid",
  "date": "2026-06-11",
  "from_warehouse_code": "00-000002",
  "to_warehouse_code": "00-000004",
  "lines": [{ "nomenclature_code": "00-00001234", "qty": 640 }]
}
```

## Служебные

### 11. `GET /ping`
Проверка связи: `{ "ok": true, "version": "1.0", "base": "БП 3.0 (3.0.74.76)" }`

## Приоритеты

| Этап | Что | Зачем |
|---|---|---|
| **1** | ping + справочники (1-5) | Заменяем Excel-выгрузки на живые данные |
| **2** | production-report + material-consumption | Главная ценность: 1С получает производство автоматически |
| **3** | payroll + stock + goods-transfer | После обкатки этапов 1-2 |
